Программа 1С. Обмен данными
О системе 1С     24.04.2017 09:04     3267



Обмен данными 1С между центром и удаленными подразделениями


Необходимость обмена данными 1С может возникать в различных случаях. Например, это требуется для работы компаний с территориально распределенной структурой, которым необходимо управлять филиалами, дочерними обществами, удаленными подразделениями и т.д. Не важно, насколько удаленными являются подразделения и филиалы, им всегда необходимо точно и вовремя подготовить консолидированную отчетность. И именно от того, как быстро и точно выполнен обмен данными 1С, зависит соблюдение сроков и трудоемкость подготовки такой отчетности. Но даже если не брать во внимание эти самые сроки и трудоемкость, то достоверность получаемых данных – обязательный фактор для принятия правильных управленческих решений.

Разноуровневая структура региональных подразделений подразумевает и организацию обмена данными 1С по различным уровням. Рассмотрим ситуацию на примере телекоммуникационной компании, имеющей подразделения в различных городах РФ. Удаленные подразделения замыкаются на центральные узлы связи (ЦУС), которые в свою очередь, в пределах одного федерального округа, подчиняются макрофилиалу.

Обмен данными 1С между информационными базами одной компании


Между различными подразделениями одной компании, не обладающей филиалами, точный и быстрый обмен данными не менее важен. Допустим, существует две или более информационные базы: база данных оперативного учета и бухгалтерская база данных. В этом случае, необходимо исключить лишнюю работу по вводу информации. Т.е. вся «первичка» должна вводиться только один раз, а в бухгалтерскую базу должны попадать уже обработанные данные, которые необходимы для отражения в бухгалтерском и налоговом учете. Это необходимо, чтобы обеспечить быстрое и точное формировании отчетности.

Организация обмена данными между различными информационными базами 1С


Обмен данными 1С осуществляется с помощью средств и решений, предоставляемых программными продуктами фирмы «1С», либо же разрабатываются специальные процедуры.

Стандартные средства позволяют создавать правила обмена, в которых указывается, какая именно информация должна попасть из одной информационной базы в другую. Такой обмен может осуществляться двумя различными способами:

1. Автоматический запуск данных по расписанию.
2. Запуск пользователем в любой момент.

Обмен осуществляется с помощью файла в формате XML, который выгружается из информационной базы (источник) и переносится в другую базу (приемник). Файл с данными возможно передать различными способами: по локальной сети, через интернет, либо через какой-либо носитель. В случае автоматизированного обмена по расписанию, необходимо, чтобы между компьютерами с базами (источником и приемником) был включен канал передачи данных.

Обмен данными можно осуществить и без использования файла с данными. В этом случае, программные продукты 1С оснащаются OLE-сервисом, который предназначен для чтения и записи информации.

Способ построения обмена данными с помощью OLE-сервиса более удобен, по сравнению с обменом через файл. В данном случае не придется тратить время на выполнение операций записи и чтения файла. Данные, таким образом, передаются быстрее и намного меньше вероятность ошибок. Но, при организации обмена информацией с помощью OLE-сервиса, необходимо обеспечить постоянный доступ между информационными базами. Необходимость постоянно поддерживать связь между компьютерами можно назвать недостатком такого варианта обмена.

Организация обмена данными между 1С и другими программами


Порой, возникает необходимость передачи данных от информационной базы 1С в совершенно другую программу и наоборот. Такой обмен информацией можно осуществить несколькими способами: через файл, либо с использованием механизмов OLE.

Распределение зарплаты по счетам затрат и формирование проводок

По завершении расчета заработной платы результаты расчета должны быть отражены на счетах бухгалтерского учета Задача распределения заработной платы по счетам затрат — одна из самых трудоемких задач учета начисленной заработной платы, находящаяся на стыке задачи расчета и задачи отражения полученных результатов в бухгалтерии Распределение заработной платы по счетам затрат, как правило, заключается в формировании бухгалтерских проводок с отражением аналитики на счетах, корреспондирующих со счетом взаиморасчетов с сотрудниками (70 в «хозрасчетном» плане счетов и 180 в «бюджетном»)

Описываемая конфигурация имеет ряд средств для учета начисленной заработной платы по счетам затрат и кодам аналитического учета

Счета

В справочнике «Счета» содержится фрагмент плана счетов предприятия, имеющий отношение к расчетам с персоналом по оплате труда Справочник «Счета» используется при вводе проводок в справочник «Хозяйственные операции»

Экспорт проводок в типовые конфигурации системы программ 1С: Предприятие. В случае экспорта проводок в типовые конфигурации системы программ 1 С Предприятие справочник «Счета» заполняется автоматически при загрузке данных об аналитическом учете из соответствующей типовой конфигурации (см пункт «Обмен данными» меню «Сервис») При этом справочник открывается только для просмотра и не доступен для редактирования

Экспорт проводок в другие бухгалтерские программы. В случае экспорта в другие бухгалтерские программы вы можете рас ширить этот справочник как новыми счетами, так и субсчетами существующих счетов

Например, в случае распределения заработной платы различных подразделений или отдельных сотрудников по субсчетам 20-го счета, необходимо ввести соответствующие субсчета и затем заполнять ими реквизиты «Дебет» и «Кредит» справочника «Хозяйственные операции»

Кроме того, на счетах можно указывать разрезы аналитического учета, в соответствии с которыми будут указываться объекты аналитического учета при описании хозяйственных операций

Справочник «Счета» редактируется достаточно редко — фактически, при первичном внедрении конфигурации — поэтому для доступа к нему не предусмотрено отдельного пункта меню Для просмотра справочника следует войти в систему с полными правами и с интерфейсом «Настройка»

Справочники «Виды аналитики» и «Элементы аналитики»

Экспорт проводок в типовые конфигурации системы программ 1 С: Предприятие. В случае экспорта проводок в типовые конфигурации системы программ 1С Предприятие справочники «Виды аналитики» и «Элементы аналитики» заполняются автоматически при загрузке данных об аналитическом учете из соответствующей типовой конфигурации (см меню «Регламенты») При этом справочники открываются только для просмотра и не доступны для редактирования

справочнике «Виды аналитики» всегда присутствуют два элемента отражающие возможность использования при формировании проводок следующих разрезов аналитического учета по сотрудникам (может использоваться на различных счетах расчетов с работниками) и по подразделениям (т е местам возникновения затрат)

Вводя в справочник «Виды аналитики» новые элементы, вы тем самым добавляете очередной разрез аналитического учета, который можно будет «подцепить» к одному из счетов в справочнике «Счета»

Справочник «Элементы аналитики», являющийся подчиненным справочнику «Виды аналитики», служит для хранения собственно произвольных объектов затрат, которые в дальнейшем могут быть использованы при описании правил распределения начисленных сумм по счетам затрат.

Хозяйственные операции

Справочник «Хозяйственные операции» содержит список хозяйственных операций, применяемых на предприятии. Вызывается справочник «Хозяйственные операции» выбором пункта «Хозяйственные операции» меню «Справочники».

Значениями из справочника «Хозяйственные операции» заполняются соответствующие реквизиты справочников «Сотрудники», «Категории» и «Подразделения», «Виды расчетов».

Каждая хозяйственная операция представляет собой корреспонденцию счетов с указанными элементами аналитики, если счета этого требуют Форма редактирования хозоперации имеет следующий вид:

В реквизит «Счет» группы «Дебет» вводится счет или субсчет дебета из справочника «Счета», в реквизите (одном или нескольких, что зависит от плана счетов программы, в которую выгружаются проводки) «Аналитический учет» записывается или выбирается из справочника «Элементы аналитики» элемент аналитического учета

Аналогичным образом заполняется группа реквизитов «Кредит», в которой заполняются счет или субсчет и элементы аналитического учета кредита.

Особым образом надо отметить счета, аналитический учет по которым ведется в разрезе сотрудников и формируется автоматически

Поэтому при использовании таких счетов ввод элементов аналитического учета запрещен

Для тех счетов, аналитический учет по которым ведется в разрезе подразделений, принято следующее соглашение' если в хозяйственной операции указано конкретное подразделение, то в проводках, формируемых по такой хозоперации будет указано именно это подразделение Если же подразделение не указано, то по такой хозоперации может быть сформировано несколько проводок с одной и той же корреспонденцией счетов. В качестве элементов аналитического учета будут указаны те подразделения, в которых работают сотрудники, чья зарплата распределяется в соответствии с этой хозоперацией.

Настройка хозоперации видов расчетов

В справочнике «Виды расчетов» хранятся, прежде всего, те виды расчетов, которые связаны с определенной хозяйственной операцией, отличной от хозяйственной операции, выбираемой по умолчанию.

Например, оплата больничного производится за счет средств соцстраха, поэтому виду расчета «Оплата больничного» будет соответствовать хозяйственная операция «За счет соцстраха» (хозяйственные операции выбираются из справочника «Хозяйственные операции»).

Порядок распределения по счетам

Собственно распределение начислений по счетам затрат производится системой при подготовке к формированию проводок При этом система просматривает все произведенные начисления и удержания и распределяет их по хозяйственным операциям по описанным ниже правилам

Каждая строка журнала расчетов «Зарплата» (см ниже) представляет собой результат выполнения определенного вида расчета (будь то начисление или удержание), проведенного для конкретного сотрудника

При формировании проводок результат каждого расчета интерпретируется как сумма по одной или нескольким хозяйственным операциям Многие документы (например, наряды, ввод начислений и удержаний, распределение основной заработной платы) могут еще во время проведения заполнять информацию о связанных с этими документами хозяйственных операциях. В любом случае информация о хозяйственных операциях анализируется при окончательном формировании проводок

Приоритетной является следующая последовательность выбора операции(й) для каждой записи журнала расчетов (если отсутствует вышестоящее условие, выполняется следующее за ним)-

Если были введены документы «Договор подряда», «Ввод начислений и удержаний» или «Распределение основной зарплаты»,  то  сумма  по  виду  расчета  распределяется пропорционально хозяйственным  операциям,  введенным  в табличных  частях  или  реквизитах  «Хозоперация»  этих документов

Если были введены другие документы с заполненным реквизитом «Хозоперация», то записи журнала расчетов сопоставляется именно эта хозоперация.

Если для вида расчета в справочнике «Виды расчетов» был установлен флажок «Распределять пропорционально расчетной базе», то сумма по виду расчета распределяется пропорционально хозяйственным операциям базовых видов расчета Такой порядок распределения   применяется   только   при   использовании расширенных возможностей расчета зарплаты и только для тех видов расчета, сумма по которым вычисляется на основе группы начислений, входящих в базу Это может быть, например, какая-либо доплата, вычисляемая в процентном соотношении от базовых начислений

Если для вида расчета в справочнике «Виды расчетов» проставлена конкретная хозяйственная операция, то выбирается именно она

Если у сотрудника в справочнике «Сотрудники» выбрана определенная хозяйственная операция, выбирается эта операция Если для подразделения, в котором числится конкретный сотрудник, введена хозяйственная операция, то выбирается данная хозоперация

Если для категории сотрудника в справочнике «Категории» проставлена хозяйственная операция, то выбирается данная хозоперация    Справочник   «Категории»   доступен    при использовании расширенных возможностей расчета зарплаты Если не были выполнены вышеперечисленные условия, то используется значение по умолчанию (константа «Хозоперация по умолчанию»)

Распределение основных начислений

Для распределения основных начислений (оклада, тарифа) по счетам затрат предназначен документ «Распределение основных начислений», который вызывается из пункта «Оплата за отработанное время» меню «Документы» Этот документ доступен только при использовании расширенных возможностей расчета зарплаты

Данный документ используется в том случае, если требуется учесть заработную плату конкретного сотрудника не по одному определенному счету и объекту аналитики, а по нескольким

При заполнении документа указывается срок действия документа (реквизиты «С» и «По»), а также суммы и хозяйственные операции, в соответствии с которыми будут в дальнейшем распределены основные начисления.

Например, если оплата сотрудника по окладу в текущем месяце составила 700 рублей, а в документе «Распределение основных начислений» указано 300 рублей по хозяйственной операции «Зарплата АУП», 500 по операции «Основное производство» и 600 — по операции «Зарплата ИТР цехов», то при формировании проводок 150 рублей будет записано на счет хозяйственной операции «Зарплата АУП», 250 на счет операции «Основное производство» и 300 — на счет операции «Зарплата ИТР цехов»

Перед началом работы с программой необходимо откорректировать справочник ставок налога на доходы физических лиц «Шкала налога на доходы», а также ставки налога те виды доходов, по которым налог исчисляется отдельно Для работы со ставками налога на доходы физлиц на закладке «Ставки» формы «Настройки» (меню «Сервис») предусмотрена кнопка «Налог на доходы физлиц»

В справочник записываются пределы доходов по мере их возрастания — сначала первый предел, потом второй и т д

В колонке «№» записывается номер предела по порядку который при добавлении новой строки в справочник формируется автоматически Этот номер можно исправить, но важно при этом помнить, что каждый последующий предел должен иметь больший номер, чем предыдущий'

В колонке «Описание» записывается наименование предела, оно носит произвольный характер, желательно отражающий суть данного предела

В колонке «Предел» записывается предельная сумма совокупного дохода, облагаемого по ставке, величина которой записывается в колонке «% налога»

Каждая строка справочника соответствует очередному пределу налогообложения, причем указанные в строке проценты используются для сумм, меньших указанного в этой же строке предела налогообложения Для правильной работы таблицы налогообложения, в послед ней строке справочника (максимальный процент налогообложения) необходимо задать очень большое число например заполнив графу «девятками»

При редактировании справочника следует помнить, что его реквизиты «Предел» «% налога > являются периодическими и хранят историю изменения своих значении Это значит что необходимо устанавливать значение реквизитов не < просто так» а на определенную дату, используя для этого кнопку «История >

Права Редактирование справочника доступно только пользователям с полными правами

Классификаторы доходов и вычетов

В соответствии с правилами формирования отчетов о доходах физических лиц и удержанном с них налоге на доходы, в налоговых карточках и справках о доходах указываются коды доходов и вычетов которые выбираются из классификаторов утвержденных МНС РФ В связи с этим в предлагаемую конфигурацию включены справочники содержащие классификаторы типов доходов типов вычетов и типов скидок

 Справочник «Доходы»

В справочнике «Доходы», кроме кода и наименования типа доходов, указано, предоставляется ли по данному типу дохода скидка, и тип скидки В справочнике «Доходы» перечислены типы доходов со гласно классификатору типов доходов, утвержденных МНС РФ

Значение реквизита «Тип скидки» заполняется значением из справочника «Типы скидок» который представлен ниже

Справочник «Типы скидок»

В справочнике <Типы скидок>, перечислены типы скидок, которые относятся к определенным доходам физических лиц согласно Инструкции ГНС №35

Справочник «Вычеты»

В справочнике «Вычеты» перечислены вычеты, предоставляемые физическим лицам согласно Инструкции ГНС №35

Обновление классификаторов «Доходы», «Вычеты» и «Типы скидок» производится автоматически при обновлении типовой конфигурации.

Важно! Алгоритмы исчисления налога на доходы физлиц и формирования отчетов о доходах опираются на кодировку типов доходов и вычетов, поэтому редактирование классификаторов типов доходов, вычетов и скидок не разрешается.

Назначение типа дохода

Назначение типа доходов конкретному начислению осуществляется в справочнике «Виды расчетов» Вызывается этот справочник вы бором пункта «Виды расчетов» меню «Справочники»

Реквизит «Тип дохода» обязательно заполняется для таких видов расчета, как подарки, призы, материальная помощь, договора гражданско-правового характера и пр Видам расчета, относящимся собственно к заработной плате (премиям, доплатам и пр. ), не следует со поставлять какого-либо типа дохода, поскольку заработная плата может относиться к различным типам дохода в зависимости от того, работает сотрудник на предприятии по совместительству или это его основное место работы

Права Редактирование справочника «Виды расчетов» доступно только пользователям с правами «главного бухгалтера»

Налоговые вычеты сотрудников

В соответствии с законодательством о налоге на доходы физические лица (в том числе работники предприятия) при налогообложении доходов, как правило, пользуются различными льготами (по терминологии Налогового Кодекса — вычетами). Так, герои Российской Федерации имеют право на ежемесячный вычет из дохода в размере 417 руб 45 коп (с 2001 года — 500 руб ежемесячно) и т д. Такие вычеты мы называем «основными», основную льготу можно установить при приеме на работу, выбрав соответствующее значение реквизита «Тип нал вычета» в документе «Приказ о приеме на работу», закладка «Общие» (по умолчанию штатным сотрудникам предлагается вычет в сумме 83 руб 49 коп или 166 руб. 98 коп (с 2001 года — 400 руб , подпункт 3 пункта 1 статьи 218 Налогового Кодекса) в зависимости от дохода за месяц, под названием «Прочие сотрудники»)

Помимо этого сотрудники, имеющие детей или других иждивенцев, пользуются вычетами на их содержание, сотрудники, являющиеся застройщиками либо покупателями жилья, пользуются правом на вычеты в размере сумм, «направленных ими на новое строительство либо приобретение жилого дома, или квартиры, или дачи, или садового домика на территории Российской Федерации» и т д. Такие дополнительные вычеты следует вводить в справочник «Налоговые вычеты» уже после приема сотрудника на работу

Необходимо отметить, что физические лица, не являющиеся штатными работниками предприятия, например совместители, также могут иметь право на предоставление вычетов при исчислении налога, которые могут быть им предоставлены по заявлению

Расчет налога на доходы физических лиц по фактически полученным доходам

Еще с 1998 года (с вступлением в силу Федерального закона №159-ФЗ от 31 12 1997) перед расчетчиками заработной платы встала проблема исчисления подоходного налога по фактически полученным доходам

В типовую конфигурацию «Зарплата+Кадры» включена возможность расчета налога по фактически полученным доходам по предложенной специалистами фирмы «1С» методике

Начисленные и полученные доходы

При исчислении налога на доходы физических лиц по фактически полученным доходам расчетчик должен знать, не только когда произведены те или иные выплаты, но и за какой период они произведены Например, известно, что некоторая сумма была выплачена физическому лицу 5 мая 1999 года в качестве заработной платы за апрель 1999 года или некоторая сумма была выплачена 20 мая 1999 года в качестве погашения долга по заработной плате за февраль 1999 года

То же самое можно сказать про любые перечисления и выплаты, производимые предприятием за физическое лицо Например, известно, что алименты, перечислены 10 мая 1999 года по результатам начисления заработной платы за апрель 1999 года.

Будем называть «начисленными» те доходы, которые начислены предприятием физическому лицу за исполнение служебных обязанностей, выполнение договорных обязательств или сложившиеся за счет получения услуг, получения материальной выгоды в любой форме и проч.

Будем называть «выплаченными» или «полученными» те доходы, которые получены физическим лицом в денежной или натуральной форме, или в форме материальной выгоды

Выбор порядка исчисления налога на доходы физических лиц

Для выбора порядка расчета налога предусмотрен переключатель в форме, вызываемой кнопкой «Налог на доходы физлиц» на закладке «Ставки» формы «Настройки» (меню «Сервис»).

В зависимости от выбранного порядка исчисления налога будет по-разному проводиться расчет налога, причитающегося к удержанию с работника, а также по-разному будут заполнены формы налоговой отчетности (справка о доходах физического лица, налоговая карточка физического лица и отчет об итоговых суммах доходов и налогов по предприятию в целом).

При расчете сумм налога, причитающегося к удержанию с работника, по полученным доходам предусмотрено два варианта учета ожидаемых к концу месяца выплат-

•  будет выплачено все, что начислено за текущий месяц;

• больше ничего выплачено не будет

Для выбора одного из этих вариантов предназначен флажок «Начисленная зарплата выплачивается в срок» на закладке «Зарплата» формы «Настройки» — установите этот флажок, если предполагается выплатить всю зарплату за текущий месяц

Здесь же указывается срок выплаты зарплаты — число, не позже которого по коллективному договору должна быть выплачена заработная плата за истекший месяц

Рекомендации по расчету налога по полученным доходам

Поскольку в доход текущего месяца включаются все выплаты через кассу (банк), то размер удержанного налога может меняться после появления в системе очередной платежной ведомости, а также, возможно, и после проведения процедуры оплаты платежной ведомости

Поэтому, если вы ежемесячно и вовремя выплачиваете зарплату, вы можете по-прежнему рассчитывать налог на доходы по начисленным суммам и только для получения налоговой отчетности по результатам закончившегося года временно включить режим расчета «по полученным доходам», при этом будет учтено возможно имевшее место округление выплачиваемых на руки сумм.

Если же на предприятии имеют место задержки выплаты зарплаты, то следует использовать расчет по полученным доходам Если вы за очередной месяц предполагаете выплатить только некоторую часть зарплаты, то для предварительного расчета ожидаемой суммы налога рекомендуется установить флажок «Начисленная зарплата текущего месяца будет выплачена» в форме группового расчета (пункт «Расчет зарплаты» меню «Регламенты»)

После того, как окончательно выяснится, что все планируемые к выплате деньги выплачены (т е все документы «Платежная ведомость» уже оплачены, а оставшиеся неоплаченными сделаны непроведенными, т е оставлены «на потом»), можно сбросить флажок «Начисленная зарплата текущего месяца будет выплачена» в форме группового расчета и пересчитать все удержания Пересчета одного подоходного налога недостаточно, поскольку от его размера зависят размеры алиментов

При этом налог на доходы будет пропорционально уменьшен по сравнению с предварительно рассчитанной суммой согласно сложившимся полученным доходам Как следствие уменьшения удержанного налога несколько увеличится переходящий на следующий месяц долг за предприятием

Правила учета доходов при расчете налога по полученным доходам

Доходом какого-либо месяца (назовем его текущий период, например февраля 2000 года) считаются суммы выплаченные через кассу (перечисленные на счета сотрудников в банках) в течение этого месяца за текущий месяц (в том числе авансы, межрасчетные выплаты, выплата под расчет за месяц) и зарегистрированные в текущем расчетном периоде документами «Платежная ведомость», как пока еще незакрытыми, так и закрытыми с датой оплаты, укладывающейся в срок выплаты зарплаты (см «Настройки», закладка «Зарплата») за текущий месяц Если ведомость оплачена позже, доход будет отложен до следующего месяца

Пример. Аванс за февраль по платежной ведомости, оплаченной 16 февраля

• Выплаченные за прошлый месяц и зарегистрированные в текущем периоде документами «Платежная ведомость» как пока еще не оплаченными, так и закрытыми с датой оплаты после срока выплаты зарплаты за прошлый месяц, но в срок выплаты зарплаты за текущий месяц

Пример. Суммы, выплаченные по платежной ведомости на выплату зарплаты за январь, оплаченной 14 февраля (срок выплаты зарплаты по коллективному договору - 8 число следующего месяца)

Суммы, выплаченные за прошлый месяц и зарегистрированные в текущем периоде закрытыми ведомостями с датой оплаты в срок выплаты зарплаты за прошлый месяц, относятся в доходы прошлого месяца

Пример. Если бы платежная ведомость из предыдущего примера была бы оплачена 8 февраля, то выплаченные по ней суммы были бы включены в доходы января, хотя сама ведомость была проведена в расчетном периоде журнала расчетов «Февраль»

•  Выплаченные за позапрошлый (или более ранний) месяц как пока еще не оплаченными ведомостями, так и закрытыми в срок выплаты зарплаты за текущий месяц ведомостями

•  Все доходы в натуральной форме (материальная выгода всех типов, питание в столовой, проездные билеты и т п ), зарегистрированные в текущем периоде

• Все удержанные из зарплаты в текущем периоде суммы

Все суммы, выплаченные по ведомостям (неважно за какой месяц), оплаченным после срока выплаты зарплаты за текущий месяц, относятся в доходы следующих месяцев

Пример. Суммы, выплаченные по платежной ведомости на выплату зарплаты за январь, оплаченной 10 марта (срок выплаты зарплаты по коллективному договору - 8 число следующего месяца) будут включены в доходы марта, хотя сама ведомость проведена и оплачена в расчетном периоде журнала расчетов «Февраль»

Кроме того, обратите внимание на следующие особенности

•  Переданные на депонент (документ «Депонирование» со сброшенным флажком «Учесть как долг за предприятием, суммы прошли по дебету счета 70(180)) суммы считаются выплаченными по тем ведомостям, по которым производилось депонирование Датой получения этих доходов считается дата оплаты соответствующей ведомости

• Суммы,   отмеченные   документом   «Депонирование»   с установленным флажком «Учесть как долг за предприятием», считаются   невыплаченными   (т к   продолжают   считаться задолженностью по зарплате)

•  Суммы возврата налога на доходы, выданные через РКО, не учитываются в составе доходов, поскольку излишне удержанный ранее налог уже был включен в совокупный доход работника

• Суммы возврата налога на доходы, зарегистрированные к выплате платежными ведомостями (т е увеличившие долг предприятия перед работником) уменьшают совокупный доход работника, так как уменьшают сумму удержанного налога (которая в свою очередь включается в совокупный доход работника)

Порядок ввода данных подробно описан ниже в разделе «Ввод данных с начала года до начала расчетов в налоговую карточку >

Возврат налога на доходы физических лиц

Документ «Возврат налога на доходы» используется для регистрации факта возврата налога на доходы Возврат налога производится в текущем расчетном периоде Для ввода данного документа выберите в меню «Документы» пункт «Налог на доходы физлиц», а затем «Воз врат налога на доходы физических лиц»

При выборе в реквизите «Сотрудник» сотрудника автоматически рассчитывается сумма переплаты налога и распределяется по видам налога (основной, с матвыгоды по вкладам и т д. ) Сумму переплаты налога можно сразу же выплатить расходным кассовым ордером или включить в очередную платежную ведомость, для этого в группе «Порядок выплаты» надо выбрать соответствующее значение

Если установлен порядок выплаты «Через РКО», то, нажав на кнопку «РКО», можно распечатать расходный кассовый ордер Сумма возвращенного налога на доходы будет учитываться при исчислении налога в текущем месяце и в последующих расчетных периодах текущего года.

Перерасчет налога по итогам года

В конфигурации имеется возможность произвести перерасчет налога на доходы по итогам истекшего отчетного года в результате различных изменений исходных данных или по фактически полученным доходам Для этого в меню «Документы» выберите пункт «Налог на доходы физлиц», а затем «Перерасчет налога на доходы физлиц»

В документе нужно определить сотрудников, для которых проводится перерасчет налога Для пересчета всех сотрудников предприятия нужно установить флаг «Все сотрудники». Если не установлен признак «Все сотрудники», то необходимо из справочника «Сотрудники» ввести тех, кому необходимо пересчитать налог на доходы Для проведения документа необходимо нажать кнопку «Провести».

По результатам перерасчета за прошлый год будет рассчитано и записано в данные сотрудника сальдо по налогу на доходы на начало текущего расчетного года, которое автоматически будет учтено при очередном расчете налога

Если при этом вы перечисляли в бюджет не всю сумму налога (которая исчислялась с начисленных доходов), а только часть этой суммы, тогда необходимо соответствующим образом скорректировать суммы налога, удержанные с сотрудников в целом за истекший год Для этого предназначены колонки удержанных сумм налога в разделе 5 налоговой карточки Для перерасчета по итогам года достаточно поправить суммы налогов в последнем заполненном месяце налоговой карточки

Налоги и отчисления во внебюджетные фонды

Помимо начисления заработной платы перед бухгалтерией стоит задача точного исчисления налогов и взносов в различные внебюджетные фонды, облагаемой базой для которых является фонд оплаты труда (ФОТ).

Налоги и перечисления

Предлагаемая конфигурация программы 1С:Предприятие содержит справочник «Налоги и перечисления», содержащий список налогов и отчислений с ФОТ предприятия в различные внебюджетные фонды, а также информацию о перечислении налога на доходы, алиментов и профвзносов. Эта информация очень редко меняется и должна быть введена в начале работы с программой. Для вызова справочника используйте пункт «Налоги и перечисления» меню «Справочники».

Справочник «Налоги и перечисления» содержит следующие сведения:

•  о фондах, в которые отчисляются налоги с фонда оплаты труда предприятия;

•  об организациях, в которые производятся отчисления с отдельных физических лиц, такие как алименты или профвзносы,

•  о  резервном фонде,  который может использоваться на предприятии, например, для оплаты отпусков

Сведения, хранящиеся в справочнике «Налоги и перечисления», используются для расчета отчислений в фонды, при формировании проводок, для формирования платежных поручений, а также для получения отчетов о перечисленных по почте алиментах

Информация о фондах и организациях вводится и редактируется в форме, внешний вид которой зависит от того, является ли данные сведения налогом или перечислением

Для налогов с ФОТ и отчислений во внебюджетные фонды форма редактирования элемента справочника имеет следующий вид

Банковские реквизиты получателей налогов (взносов) заносятся в группе реквизитов «Получатель» на закладке «Банковские реквизиты»

На закладке «Расчетные» указываются процент налога (отчисления) и регистрационный номер предприятия в фонде (ГНИ)

В группе реквизитов «Бухгалтерские счета» для налогов (взносов), включаемых в себестоимость продукции, следует установить флажок «Включается в себестоимость» и указать кредитуемый счет, на котором учитываются расчеты по налогу или с внебюджетным фондом, реквизит «Дебетуемый» для них должен остается незаполненным, поскольку эти затраты распределяются по тем же счетам, что и суммы оплаты труда Для налогов, относящихся на прибыль, например, сбора на развитие образовательных учреждений, флажок «Включается в себестоимость» следует сбросить и указать кроме кредитуемого счета также и дебетуемый счет, фактически один из субсчетов 80-го балансового счета

Кнопка «База .» позволяет отредактировать облагаемую базу налога или отчисления (см ниже)

Для различных перечислений форма редактирования имеет следующий вид

Здесь следует заполнить только банковские реквизиты получателя перечисления

Если на хозрасчетном предприятии формируются резервные фонды (например, резерв отпусков), то в этот справочник следует ввести строку с наименованием «Резерв» и проставить процент отчисления в резервный фонд

В случае использования предлагаемой конфигурации в бюджетных организациях для «лишних» строки можно установить нулевое значение реквизита «Процент», либо вовсе их удалить

Важно! Обязательно проверьте соответствие счетов, занесенных в реквизиты группы «Бухгалтерские счета» справочника «Налоги с предприятия», тому плану счетов, который используется на предприятии, и, если нужно, отредактируйте их.

Права Редактирование справочника доступно только пользователям с полными правами.

Облагаемая база для налогов и взносов

Редактирование состава облагаемой базы описано в пункте «На значение базы начислении» раздела «Виды расчетов»

Импорт адресных классификаторов

Данная  процедура  предназначена для  импорта  адресных классификаторов, используемых при вводе различных адресов (пункт «Импорт» меню «Файл»)

Импорт классификаторов может потребоваться как в начале эксплуатации программы, так и в случае их обновления

В поле «Откуда» надо выбрать дисковод, в котором находится дискетка с обновленными классификаторами, либо выбрать каталог, в котором находятся файлы с классификаторами из ГНИ

Кроме того, необходимо указать вид классификатора, который предполагается обновить Для этого следует щелкнуть мышью в окошке рядом с названием классификатора

В классификаторы, имеющиеся в информационной базе, добавляются новые значения из загружаемого классификатора Если установить флажок «Заменять старые наименования существующих объектов», то в этом случае при загрузке программа будет заменять старые значения, если они отличаются от новых Это может быть необходимо, если, например, в адресном классификаторе поменялись названия районов или городов

Если же установить флажок «Полностью очищать классификаторы перед загрузкой», то перед загрузкой новых значений имевшиеся в информационной базе классификаторы будут удалены, что требует значительных затрат времени

Чтобы начать загрузку классификаторов, следует нажать кнопку «Импорт», после чего процесс загрузки отразится в строке «Состояние»

Импорт из 1С: Налогоплательщика

Если   на   предприятии   ранее   применялась   программа «1С'Налогоплательщик» версий 7 5 или 7 7, то список сотрудников и данные о доходах из этой программы можно импортировать в информационную базу конфигурации «Зарплата + Кадры» (пункт «Импорт» меню «Файл»)

После указания имени файла (или каталога с программой «1С Налогоплательщик 75») нажмите кнопку «Импорт» Процесс загрузки отразится в строке «Состояние»

В результате импорта список физических лиц из программы «1С'Налогоплательщик» будет перенесен в справочник «Сотрудники», а данные о введенных доходах будут перенесены в налоговые карточки

Загрузка данных из текстового файла

Данная процедура предназначена для загрузки в информационную базу данных из текстового файла Текстовый файл, из которого загружаются данные, должен описывать данные в формате, опубликованном диске ИТС Например, с помощью этой процедуры загружаются данные об используемом плане счетов, выгруженные в текстовый файл конфигурациями «Бухгалтерский учет» (начиная с редакции 3 5), «Производство+Услуги+Бухгалтерия» (начиная с редакции 1 5) и Комплексной конфигурацией (начиная с редакции 2 5)

Процедура «Загрузка данных» вызывается из пункта «Обмен данными» меню «Сервис» главного меню программы

В диалоге необходимо указать имя файла загрузки — это имя текстового файла, откуда будут загружены данные

Кнопка «Выбрать» открывает каталог, в котором находится загружаемый файл

По кнопке «Загрузить» начинается загрузка объектов из файла в информационную базу Если установлен флажок «Комментировать процесс загрузки в окне сообщений» сообщения о ходе загрузки будут выводиться в табло

Ошибки, выявленные при разборе файла загрузки, будут записаны в файл Errors txt в каталоге, откуда запущена обработка загрузки данных

Если установлен флажок «Сформировать отчет о результатах загрузки», то после загрузки данных будет открыт отчет, содержащий перечень загруженных справочников и документов:

Колонки отчета

• «№ строки» — номер строки файла загрузки,      описывающей загруженный объект;

• «Строка» — содержание строки файла загрузки,            описывающей загруженный объект;

• «Вид объекта» — тип и вид загруженного объекта,

• «Объект» — загруженный объект;

• «Результат» — результат загрузки- был найден существующий объект или создан новый.

Перерасчет окладов и тарифов

Регламентная процедура «Перерасчет окладов/тарифов» предназначена для изменения на определенный коэффициент величин, образующих основной заработок: оклады, тарифы, нормы сдельных расценок (пункт «Перерасчет окладов» меню «Регламенты»)

Изменяемые величины являются реквизитами справочников типовой конфигурации и являются периодическими, т.е. программа сохраняет всю историю их изменений. В реквизите «Пересчитать на дату» надо задать дату обновления изменяемой величины (по умолчанию заполняется рабочей датой, см. пункт «Параметры» меню «Сервис», закладка «Общие»)

Опции группы «Обновить данные в справочниках» позволяют управлять объемом изменяемой информации. Так например, при изменении окладов и тарифов на всем предприятии необходимо пропорционально изменить все расчетные данные (для этого нужно установить все флажки)

В результате проведения данной процедуры можно изменить на заданный коэффициент следующие значения

•  оклады в справочниках «Сотрудники», «Должности», «Штатное расписание»;

•  исходные данные для расчета окладов по ETC — константу «Тариф 1-го разряда», значения графы «Доплата» в справочнике «Тарифная сетка»,

•  значения графы «Тариф» в справочнике «Разряды рабочих»;

•  сдельные расценки в справочнике «Расценки».

Если требуется изменить оклады сотрудникам только одного подразделения, тогда при установленных флажках группы «Сотрудников» можно выбрать реквизит «Подразделение».

Важно! В отличие от документов типа «Кадровое перемещение», которым также можно оформить изменение окладов/тарифов, автоматически отменить результаты перерасчета окладов, тарифов и других реквизитов невозможно.

Замечание. Если требуется изменить только оклады сотрудников, при этом на предприятии используется штатное расписание (см установки в стартовом Помощнике или форме «Настройки»), то помимо справочника «Сотрудники» не забудьте изменить оклады и в справочнике «Штатное расписание».

Работа с периодическими реквизитами справочников

В формах списков и формах редактирования элементов справочников, имеющих периодические реквизиты, присутствует кнопка «История», которая позволяет вывести для просмотра и, в некоторых случаях, для редактирования всю историю всех периодических реквизитов выбранного элемента справочника (того элемента, который выделен в списке справочника).

При вызове истории значений из формы элемента справочника редактирование ее не разрешается

Кроме того, в форме элемента справочника «Сотрудники», в которой редактируется большое количество периодических реквизитов, присутствуют кнопки , расположенные в непосредственной близости от одного или нескольких периодических реквизитов Нажав на эту кнопку, можно просмотреть историю значений только этой группы реквизитов (или одного реквизита)

В некоторых формах списков справочников, которые содержат периодические реквизиты, существует возможность менять дату их просмотра и редактирования (см например, справочник «Валюты») Для этого в форме списка справочника предназначена кнопка выбора даты просмотра периодических реквизитов Изменение даты приводит к обновлению всех отображаемых значений на эту дату При редактировании значения такого реквизита в списке справочника значение будет редактироваться на указанную дату

Просмотр истории периодических реквизитов

При нажатии кнопки «История» на экран будет выдано окно с таблицей, в которой колонки содержат значения периодических реквизитов, а строки соответствуют датам, на которые вводились эти значения В заголовке окна в круглых скобках выводится наименование элемента справочника, для которого вызвано окно истории

В колонке «Дата» показывается дата, на которую установлено значение реквизита.

Если хотя бы один реквизит справочника может редактироваться документами, то после колонки «Дата» в таблице будет выводиться колонка «Документ», в которой указывается документ, вызвавший изменение значения реквизита.

Далее следуют колонки со значениями периодических реквизитов справочника Каждому изменению значения соответствует строка с указанием даты, документа, изменившего значение реквизита (если такой документ есть), и собственно измененного значения

Обратите внимание, что в колонке «Дата» содержатся все даты, на который изменялось значение хотя бы одного периодического реквизита справочника Значение того реквизита, который был изменен на эту дату, будет располагаться в колонке, соответствующей данному реквизиту, ячейки в остальных колонках будут пустыми

Если на одну дату (один документ) приходится изменение нескольких реквизитов, то все измененные значения показываются в «своих» колонках в одной строке таблицы

Редактирование истории периодических реквизитов

В нижней части окна истории периодических реквизитов располагаются кнопки для редактирования значений реквизитов

Ввод нового значения реквизита. Для ввода нового значения периодического реквизита служит кнопка «Добавить» При ее нажатии вызывается диалог редактирования значения реквизита Внешний вид диалога может незначительно различаться в зависимости от типа редактируемого значения

В этом диалоге следует указать дату, на которую необходимо ввести новое значение реквизита, и само значение После нажатия кнопки «ОК» в историю значения реквизита будет добавлена новая строка

Если при вводе нового значения будет указана дата, уже существующая в истории значения реквизита, то старое значение реквизита будет заменено на указанное новое значение Однако, если значение на указанную дату ранее было введено документом, то в истории значения появится еще одна строка — с указанной датой и значением. Но значение, введенное вручную, будет располагаться перед значениями, введенными документами.

Изменение значения реквизита. Кнопка «Изменить» позволяет изменить значение активного реквизита на дату, которая соответствует текущей позиции курсора. При нажатии на эту кнопку будет вызван диалог, аналогичным тому, который используется при добавлении значения реквизита.

Удаление значения реквизита. Кнопка «Удалить» удаляет значение активного реквизита на дату, соответствующую позиции курсора. Если активный реквизит является единственным реквизитом на эту дату, то удаляется и сама запись истории.

Кнопка «Удалить на дату» удаляет значения всех реквизитов на дату, соответствующую позиции курсора, а также саму запись истории.

Комментарий к документам

На практике часто случается, что помимо заполнения обязательных реквизитов документов есть необходимость сделать как бы «пометки на полях» документа То есть записать в сам документ какую-либо произвольную информацию, например, инструкции о том, что еще нужно сделать с документом.

Для решения этой проблемы во всех документах типовой конфигурации предусмотрено поле комментария, которое может выглядеть как обычная строка внизу формы документа: или располагаться на отдельной закладке-

Блокнот

При постоянной работе с программой удобно посылать себе сообщения о том, что сегодня надо сделать и о чем не забыть. Такой возможностью обладает включенный в описываемую конфигурацию

 «Блокнот» Вызвать список напоминаний, хранящихся в «Блокноте», можно, нажав кнопку  панели инструментов

Нажав на кнопку «Новое» в списке напоминаний, можно ввести новое напоминание, кнопкой «Удалить» можно удалить отмеченное напоминание, нажав на кнопку «Изменить» можно отредактировать отмеченное напоминание

Первая и вторая колонки списка напоминаний представляют собой дату и время очередного появления соответствующего напоминания на экране Символ третьей колонки показывает, как часто будет появляться напоминание Р - разовое, Д - ежедневное, Н - еженедельное, М - ежемесячное и К - ежеквартальное Последняя колонка содержит краткое содержание напоминания

Рассмотрим назначение реквизитов формы при нажатии на кнопку «Изменить», предназначенной для редактирования напоминания

В поле «Комментарий» вводится краткое содержание сообщения Краткое содержание выводится в списке напоминаний в последней колонке. В поле сообщения можно ввести текст произвольной длины

Опция «Периодическое» определяет периодическое это напоминание (флажок установлен) или разовое Разовое напоминание появляется на экране один раз тогда, когда это определено реквизитами «Дата» и «Время» Частота появления периодического напоминания определяется значением реквизита «Периодичность» и варьируется от ежедневных напоминаний до ежеквартальных

Ежедневные напоминания появляются на экране каждый день (конечно, если при этом запускается система 1С:Предприятие) тогда, когда это определено реквизитом «Время»

Еженедельные напоминания появляются на экране каждую неделю по дням, определяемым реквизитом «Дата» Например, еженедельное напоминание, введенное в пятницу, 10 октября 1997 года, будет появляться на экране по пятницам

Аналогично, ежемесячные (ежеквартальные) напоминания появляются на экране в один и тот день очередного месяца (квартала) Например, ежемесячное напоминание, введенное 10 октября 1997 года, в следующий раз появится на экране 10 ноября 1997 года.

Напоминание выводится при работе программы в окне следующего вида

После появления на экране можно вывести это же напоминание повторно через произвольный интервал времени от 1 минуты до 1 часа. Для этого следует установить флажок «Повторить еще» и ввести нужный интервал времени

Правовая поддержка

При работе с конфигурацией у пользователя могут возникать вопросы, связанные с нормативными документами, на основании которых реализованы многие расчеты, формирование отчетности, ввод первичных документов и т д. Для этого имеется возможность вызова правовой системы «1С Гарант. Правовая поддержка» при нажатии кнопки  панели инструментов, если эта система установлена на рабочем компьютере. Если правовая система не установлена, то выдается следующее сообщение.

Если правовая система установлена, то во многих документах, справочниках и отчетах появляется кнопка вызова, при нажатии на которую вызываются необходимые нормативные документы, относящиеся к конкретному действию. Например, кнопка  в документах «Оплата сверхурочного времени» и «Приказ о работе в праздничные и выходные дни» вызывает одинаковый нормативный документ; в документе «Больничный лист» при нажатии соответствующей кнопки отображается пакет нормативных документов, относящихся к начислению страховых взносов и т.д.

Обновление конфигурации

Часто возникает необходимость изменения типовой конфигурации, если происходят изменения в законодательстве или по любым другим причинам. В этом случае необходимо сделать обновление текущей рабочей конфигурации, при этом данные сохраняются. Обновление производится в режиме конфигурирования информационной базы, т.е. при запуске системы «1С:Предприятие» в режиме «Конфигуратор».

Для обновления релиза конфигурации следует использовать режим «Объединение конфигураций» Вам потребуется файл метаданных (файл lCv7.MD) нового релиза конфигурации Для этого установите новый релиз конфигурации в отдельный каталог жесткого диска (по окончании процесса обновления этот каталог можно удалить).

1. Запустите систему «1С:Предприятие» в режиме «Конфигуратор».

2. Сделайте архивную копию вашей информационной базы

3. Вызовите режим «Объединение конфигураций», для этого в меню «Конфигурация» выберите пункт «Объединение конфигураций».

4. В запросе «Открыть файл конфигурации» выберите файл метаданных lCv7.MD нового релиза конфигурации

5. В окне режима «Объединение конфигураций» проверьте:

•  в группе «Приоритет конфигурации» установлен переключатель «Загружаемая конфигурация»,

•  в группе «Метод объединения» установлен переключатель «Замещать объекты».

6. Нажмите кнопку «ОК» для объединения конфигураций.

7. После завершения будет открыто окно «Конфигурация», содержащее конфигурацию с внесенными изменениями. Выполните сохранение конфигурации.

Подробные описание изменений в типовой конфигурации можно получить на дисках Информационно-технологического сопровождения 1С:Предприятия (диски ИТС) или на линии консультаций фирмы «1С» и ее партнеров.

Календари 2-ой смены сдвинуты на один день: основной — 0, 24, 0, 0; вечерний — 0, 2, О, О и ночной — О, 8, 0, 0. Календари третьей и четвертой смен формируются аналогично.

Методика конвертирования данных программы 1С .'Зарплата для DOS

Система 1 С Зарплата и Кадры, как одна из реализации системы 1С Предприятие, является совершенно новым продуктом, не имеющим программ-предшественников, созданных на других платформах Тем не менее, система содержит встроенное средство, позволяющее облегчить перенос данных программы 1С Зарплата для DOS, вне зависимости от того, какие предметные задачи решались в программе 1С Зарплата

Конвертирование, разумеется, производится в режиме конфигурирования информационной базы, т е при запуске системы в режиме «Конфигуратор» При этом результаты конвертирования, т е новые справочники, перенесенные из 1 С Зарплаты, будут введены в текущую информационную базу

Конвертер данных системы 1 С Предприятие позволит перенести данные из справочников программы lC-Зарплата Реализация в программе lC-Зарплата принципа хранения «свернутых» архивных данных приводит к тому, что при помощи стандартного конвертера данных не возможен адекватный перенос информации из архивов этой программы в информационную базу 1С'Предприятия

С другой стороны, используя возможности встроенного языка системы 1С:Предприятие, предназначенные для работы с внешними Xbase-таблицами, можно создать процедуру переноса архивных данных 1С:Зарплаты в информационную базу 1С:Предприятия.

Создание такой процедуры требует определенной квалификации от администратора системы 1С:Предприятие, знания конфигурации «Расчет зарплаты и Кадровый учет», описанной в данном руководстве и, разумеется, хорошего знания структуры данных конкретной настройки программы 1С:Зарплата, данные которой необходимо перенести.

Ниже, в разделе «Рекомендации» приведен ряд общих замечаний, которые помогут вам создать такого рода процедуру переноса не только справочных данных, но и архивов вашей прежней системы расчета заработной платы.

Важно. Стандартный конвертер данных программы 1С:Зарплата позволит вам перенести информацию всех справочников программы, но не предназначен для переноса архивных данных расчета.

Ниже изложена информация, предполагающая хорошее знание программы 1С:Зарплата. Большинство терминов этой программы приводятся без дополнительных пояснений.

Конвертер «умеет» конвертировать данные программы 1С:Зарплата версий 3.0, 3.1 и 4.0 и вызывается пунктом «Конвертировать данные...» меню «Сервис» при работе в режиме конфигурирования.

При переносе содержимого всех справочников программы 1С:Зарплата производятся следующие действия.

•  Конвертирование, т.е. перенос данных в информационную базу, производится полуавтоматически, т.е. конвертер пытается, основываясь  на  названиях  справочников  1С:Зарплаты  и идентификаторах уже существующих справочников текущей информационной базы, установить соответствие между ними и предлагает пользователю подтвердить или отвергнуть найденные соответствия.

•  Основной справочник программы 1С:Зарплата (как правило, это справочник «Сотрудники») конвертируется в двухуровневый справочник системы 1С:Предприятие. При этом группы элементов справочника (папки) на первом уровне соответствуют ведомостям программы 1С:Зарплата, а собственно, элементы справочника соответствуют сотрудникам.

• Локальные справочники программы 1С:Зарплата, которые связаны с основным справочником по его коду, конвертируются в подчиненные справочники системы 1С:Предприятие. Например, справочник «Дети сотрудников» программы 1С:Зарплата, который является локальным и имеет ключевое выражение типа «Сотрудники:ТабНомер», будет конвертирован в справочник, подчиненный справочнику «Сотрудники» в рамках системы 1С:Предприятие.

• Общие справочники программы 1С:Зарплата конвертируются в обычные, одноуровневые справочники системы 1С:Предприятие.

•  В процессе конвертирования конвертер разбирает взаимосвязи справочников программы 1С:Зарплата и на основании набора ключевых выражений каждого справочника устанавливает тип данных связанных с ним граф других справочников. Это приводит к тому, что реквизиты вновь создаваемых справочников могут иметь типы данных, отличные от базовых («число», «строка» и «дата»). (т.н. «связанные макроимена»), то при конвертировании эти графы получат тип «Справочник» конкретного типа «Банки».

Выбор настройки. Процесс конвертирования начинается с выбора конвертируемой настройки программы 1С Зарплата При этом конвертер предлагает выбрать каталог с так называемой типовой настройкой программы 1С Зарплата Этот каталог распознается конвертером по наличию файла-описателя Config tn, который содержит описание типовой настройки, и, как правило, находится в подкаталогах с расширением имени 1SZ каталога 1SZ 40, 1SZ 31 или 1SZ 30 (в зависимости от версии программы 1С Зарплата для DOS, например, \1SZ40\RFBASIC 1SZ)

После выбора настройки конвертер пытается проанализировать содержимое настройки и определить версию программы 1С Зарплата

В случае, если чтение и разбор файла-описателя прошли удачно, конвертер считывает данные о типовой настройке программы 1С Зарплата и выдает основную панель

Использование второй графы как наименования. В программе 1С Зарплата все графы справочников, кроме первой (ключевой) графы, равноправны Тем не менее, на практике часто вторая графа справочника используется как описание или наименование соответствующего объекта — наименование категории работника, наименование подразделения и пр. Для того, чтобы превратить вторую графу справочника 1С Зарплаты в «Наименование» справочника текущей информационной базы, используйте параметр «Интерпретировать вторую графу как Наименование» При этом все графы справочника 1С Зарплата будут конвертированы как реквизиты справочника 1С Предприятия, первая графа будет интерпретирована как код, а вторая — как наименование

Управление списком реквизитов. Если конвертер предлагает использовать уже существующий справочник, панель выводится в «развернутом» виде — со списком реквизитов Если конвертер не может предложить справочник по умолчанию, панель выводится без списка реквизитов, причем по умолчанию справочник предлагается не конвертировать

После этого можно, тем не менее, задать необходимость конвертирования справочника и даже указать соответствие его уже существующему справочнику текущей конфигурации 1С Предприятия

После того, как в поле «Использовать существующий справочник» будет что-нибудь выбрано, программа попытается установить соответствие граф старого справочника и реквизитов нового и выведет панель в «развернутом» виде Пользуясь кнопками «Добавить связь» и «Удалить связь» можно управлять списком соответствия реквизитов справочников При этом, при конвертировании основного справочника программы 1С Зарплата (как правило, «Сотрудники») связь можно устанавливать для всех граф старого справочника, а при конвертировании прочих справочников связь можно устанавливать для всех реквизитов кроме первого (в случае, если вторая графа не используется как «наименование») или кроме первого и второго (в случае, если вторая графа используется как «наименование»)

Для установки очередной связи в списке реквизитов следует выбрать строку с реквизитом, еще не имеющим связи (т е для которого еще не установлено соответствие реквизиту нового справочника) и нажать кнопку «Добавить связь». При этом появится следующее окно, в котором можно выбрать реквизит существующего справочника системы 1С Предприятие

Для удаления связи в списке реквизитов следует выбрать строку с реквизитом, для которого уже установлено соответствие реквизиту нового справочника, и нажать кнопку «Удалить связь»

При редактировании списка реквизитов можно отказаться от конвертирования той или иной графы справочника 1С Зарплаты, убрав значок </ рядом с названием графы Это необходимо в том случае, если данные справочника «идеологически» не вписываются в текущую конфигурацию системы 1С Предприятие Например, графы справочника «Сотрудники», содержащие налогооблагаемую сумму за год, не используются в описываемой конфигурации lC-Предприятия

Конвертирование. После выбора действия «Конвертировать», программа пытается интерпретировать установленные параметры переноса данных, провести преобразование типов данных, если это необходимо, и установить связи между справочниками прежней программы 1С Зарплата При этом например, если в прежней программе справочник «УченыеЗвания» использовался для заполнения графы «Звание» справочника «Сотрудники», это будет переведено в термины 1С Предприятия и в информационной базе появится справочник «УченыеЗвания», а в новом справочнике сотрудников появится реквизит типа «справочник УченыеЗвания»

Если при конвертировании в настройке программы 1С Зарплата обнаруживаются несовпадения описателя настройки и реальных данных, конвертер выдаст сообщение о невозможности открыть файлы настройки программы 1С Зарплата

Рекомендации

Перед конвертированием данных следует изучить характеристики описываемой в данном руководстве конфигурации Это поможет продумать, какие именно справочники следует переносить в текущую конфигурацию, а какие нет Мало перенести содержимое справочников в текущую конфигурацию, необходимо заставить их «работать» В связи с тем, что концепции программ 1С Зарплата и 1С Предприятие слишком далеки друг от друга, многие «зарплатные» данные просто не имеют смысла в текущей конфигурации Примером могут служить справочники перечислений (список сотрудников, перечисляющих заработок в банки) и алиментщиков (список сотрудников, оплачивающих исполнительные листы) для которых, как правило, «не находится места» в «зарплатной» конфигурации 1С Предприятия Такого рода данные, как правило, вводятся в систему через документы, т е объекты, не имеющие аналога в программе 1С Зарплата      "

Очевидно, перенос основного справочника программы 1С Зарплата имеет смысл, перенос таких «маленьких» и легко интерпретируемых справочников как «Категории», «Должности» также имеет смысл

Обработка данных после конвертирования. В том случае, если упомянутые выше справочники перечислений и алиментов содержат много информации их также можно конвертировать, но не для прямого использования, а для автоматического ввода на их основе соответствующих документов системы 1С Предприятие

Например, после конвертирования данных вы можете создать процедуру на встроенном языке программы 1С Предприятие (например, как модуль обработки), которая будет просматривать новые справочники, формируя на основании находящихся в них данных соответствующие документы (такие как «Исполнительный лист» или «Перечисление зарплаты в банк»)

Таким образом, если программа 1С Зарплата содержит очень большое количество данных, возможен их полный перенос в текущую конфигурацию с тем, чтобы некоторые из справочников использовать непосредственно, а некоторые использовать как исходный материал для преобразования в документы 1С Предприятия средствами встроенного языка

Перенос архивных данных. Конвертирование так называемых архивных данных программы 1С Зарплата требует особого обсуждения

Дело в том, что принцип хранения результатов расчетов предыдущих месяцев в свернутом виде, который применяется в программе 1С Зарплата и который продиктован «экономностью» программы с точки зрения расходуемого дискового пространства и скорости доступа к данным, не позволяет перенести архивные данные непосредственно

Компонента «Расчет» системы программ 1 С Предприятие реализует другой подход к хранению данных — хранение полной копии всех результатов расчета в журналах расчетов. И хотя можно провести аналогию между содержимым окна «Данные» программы 1С Зарплата и журналами расчетов 1С Предприятия, эти данные не накапливались во времени, а сворачивались по тем или иным принципам в так называемые «архивные» файлы

Свертка результатов отдельных видов расчета в одно значение (например, «все начисления», «налогооблагаемая сумма» и тд) исключает возможность провести обратный процесс — разобрать полученные данные на составляющие — результаты конкретных расчетов

В связи со всем вышесказанным, стандартный конвертер не способен провести перенос архивных данных Тем не менее, возможна обработка этих данных с целью переноса наиболее полезной информации и интерпретации их в терминах настоящей конфигурации «Расчет Зарплаты и Кадровый учет»

Все данные программы 1С Зарплата хранятся в DBF-формате. Встроенный язык программы 1С Предприятие поддерживает работу с внешними данными в этом формате (т н Xbase-объекты) Это позволяет написать процедуру обработки таких данных и перенос той или иной информации в существующие справочники конфигурации

Рассмотрим, например, вопрос переноса данных для расчета налога на доходы в текущую конфигурацию Необходимая информация, как правило находится в тех или иных графах архивных файлов программы 1С Зарплата Такие данные, как сумма к налогообложению, размер удержанного пенсионного фонда и пр. могут быть перенесены в соответствующие реквизиты справочника сотрудников информационной базы 1С Предприятия Для этого необходимо создать набор Xbase-объектов (соответствующих архивным файлам 1С Зарплаты за все месяцы) с одной стороны, и объект «Справочник Сотрудники» с другой стороны Перебрав все архивные данные и просуммировав необходимую информацию, необходимо заполнить ее в соответствующие реквизиты справочника сотрудников При этом необходимо пользоваться содержимым графы с именем сотрудника в архивных данных как ключом при поиске конкретного элемента справочника сотрудников по наименованию

Названия архивных файлов программы 1С Зарплата формируются следующим образом первый символ — последняя цифра года, например, для 1996 — 6, второй символ — номер месяца, причем десятому, одиннадцатому и двенадцатому месяцам соответствуют символы А, В и С; остальные символы — название ведомости программы 1С Зарплата Эти названия следует использовать как параметр метода ОткрытьФайл Xbase-объекта

Ошибки открытия данных 1С: Зарплаты. Очень важно перед конвертированием убедится в том, что настройка программы 1С Зарплата не содержит внутренних дефектов с точки зрения конвертера данных Это связано с тем, что программа 1С Зарплата значительно «либеральнее» к описанию структур справочников, например, вполне допустимы графы справочников без названий' неполное описания справочника и т д. Кроме того, описание структур справочников (файл Attnb dat программы 1С.Зарплата) и их реальная структура могут не совпадать в результате не до конца проведенной реструктуризации,   что   «на   ходу»   обрабатывается   программой 1С Зарплата Все это связано с изменчивостью структур данных программы в разных ее реализациях, использованием настроек, созданных в более старых реализациях программы, новыми реализациями и тд

Во всяком случае, при возникновении сообщений об ошибках чтения данных 1С:Зарплаты, следует восстановить целостность данных программы. Например, для того, чтобы структура справочников соответствовала описателю данных 1С:Зарплаты в формате текущей версии программы, необходимо просто открыть для редактирования структуру соответствующего справочника внести в нее не затрагивающие сути изменения (например, увеличив размер графы на 1) для того, чтобы вызвать принудительную перестройку справочника и перезапись его структуры в файл Attrib dat.

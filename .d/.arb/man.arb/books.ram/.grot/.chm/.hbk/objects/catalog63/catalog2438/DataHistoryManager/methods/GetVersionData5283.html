МенеджерИсторииДанных.ПолучитьДанныеВерсии (DataHistoryManager.GetVersionData)

МенеджерИсторииДанных (DataHistoryManager)

ПолучитьДанныеВерсии (GetVersionData)

Доступен, начиная с версии 8.3.11.

Синтаксис:

ПолучитьДанныеВерсии(<Данные>, <НомерВерсии>)

Параметры:

<Данные> (обязательный)

Тип: СправочникСсылка.<Имя справочника>, ДокументСсылка.<Имя документа>, РегистрСведенийКлючЗаписи.<Имя регистра сведений>, БизнесПроцессСсылка.<Имя бизнес-процесса>, ЗадачаСсылка.<Имя задачи>, ПланВидовХарактеристикСсылка.<Имя плана видов характеристик>, ПланСчетовСсылка.<Имя плана счетов>, ПланОбменаСсылка.<Имя плана обмена>, ПланВидовРасчетаСсылка.<Имя плана видов расчета>, КонстантаКлючЗначения.<Имя константы>.
Данные, по которым создана версия.

<НомерВерсии> (обязательный)

Тип: Число.
Номер версии.

Возвращаемое значение:

Тип: Структура.
Формируемая структура содержит все поля и табличные части версии. Для полей структура содержит:
  • Ключ - имя поля.
  • Значение - значение поля. Ссылки предоставляются в виде структуры, описанной ниже.
Для табличных частей структура содержит:
  • Ключ - имя табличной части.
  • Значение - тип ТаблицаЗначений. Данные табличной части.
Структура для ссылок:
  • Ссылка (Ref) - тип Ссылка на объект информационной базы. Ссылка на объект.
  • Представление (Presentation) - тип Строка. Представление ссылки.
Внимание! Возвращаемое значение не содержит независимые разделители и их значения.

Возвращает Неопределено, если версия создана по удалению данных.

Описание:

Возвращает данные версии объекта.

Доступность:

Сервер, толстый клиент, внешнее соединение.
Вызов метода выполняет обращение к серверу.

Пример:

ДанныеВерсии = ИсторияДанных.ПолучитьДанныеВерсии(
    Справочники.Валюты.НайтиПоНаименованию("Рубль"), 1);

Использование в версии:

Доступен, начиная с версии 8.3.11.

ТекстовыйДокумент (TextDocument)

ЗаписатьАсинх (WriteAsync)

Доступен, начиная с версии 8.3.18.

Вариант синтаксиса: В файл

Синтаксис:

ЗаписатьАсинх(<ИмяФайла>, <ТипФайла>, <РазделительСтрок>)

Параметры:

<ИмяФайла> (обязательный)

Тип: Строка.
Имя файла для сохранения текста.

<ТипФайла> (необязательный)

Тип: КодировкаТекста, Строка.
Тип кодировки файла. Тип файла может быть задан как в виде значения перечисления КодировкаТекста, так и в виде строки с указанием названия кодировки.

Значение по умолчанию: UTF8.

<РазделительСтрок> (необязательный)

Тип: Строка.
Задается строка, являющаяся разделителем строк, к которой при записи файла будет преобразован символ ПС. Если строка пустая, то разделители строк при записи не конвертируются.
Допустимые варианты разделителей строк:
  • ВК;
  • ПС;
  • ВК + ПС (значение по умолчанию - может не указываться).
При использовании других строк в качестве разделителей будет вызвано исключение.
Значение по умолчанию: ВК + ПС.

Описание варианта метода:

Запись текстового документа в файл.

Вариант синтаксиса: В поток

Синтаксис:

ЗаписатьАсинх(<Поток>, <ТипФайла>, <РазделительСтрок>, <ЗаписатьBOM>)

<Поток> (обязательный)

Тип: Поток, ПотокВПамяти, ФайловыйПоток.
Поток для записи документа.

<ТипФайла> (необязательный)

Тип: КодировкаТекста, Строка.
Тип кодировки текста. Тип кодировки может быть задан как в виде значения перечисления КодировкаТекста, так и в виде строки с указанием названия кодировки.

<РазделительСтрок> (необязательный)

Тип: Строка.
Задается строка, являющаяся разделителем строк, из которой при чтении файла будет сформирован символ ПС.
Если строка пустая, то разделители строк при чтении не конвертируются.
Допустимые варианты разделителей строк:
  • ВК;
  • ПС;
  • ВК + ПС (значение по умолчанию - может не указываться).
При использовании других строк в качестве разделителей будет вызвано исключение.
Значение по умолчанию: ВК + ПС.

<ЗаписатьBOM> (необязательный)

Тип: Булево.
Определяет, будет ли добавлена метка порядка байт (BOM) при сохранении документа.
Значение по умолчанию: Ложь.

Описание варианта метода:

Запись текстового документа в поток.

Возвращаемое значение:

Тип: Обещание.
Результат выполнения обещания:
  • Истина - если запись была инициирована с помощью расширения браузера и пользователь согласился на его получение, то есть запись файла была завершена.
  • Ложь - если запись была инициирована с помощью расширения браузера, но пользователь не согласился на его получение, то есть запись файла не была завершена.
  • Неопределено - если запись была инициирована скачиванием файла средствами браузера и неизвестно, был ли файл записан или нет.

Описание:

Асинхронная функция. Записывает текст в файл с именем <ИмяФайла>.

Доступность:

Тонкий клиент, веб-клиент, толстый клиент.

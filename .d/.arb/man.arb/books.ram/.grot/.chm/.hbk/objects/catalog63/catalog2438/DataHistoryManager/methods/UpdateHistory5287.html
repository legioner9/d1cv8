МенеджерИсторииДанных.ОбновитьИсторию (DataHistoryManager.UpdateHistory)

МенеджерИсторииДанных (DataHistoryManager)

ОбновитьИсторию (UpdateHistory)

Доступен, начиная с версии 8.3.10.

Вариант синтаксиса: Обновление истории данных

Синтаксис:

ОбновитьИсторию(<ВыполнитьОбработкуПослеЗаписиВерсий>, <АвтоУдалениеИзОбработкиПослеЗаписиВерсий>)

Параметры:

<ВыполнитьОбработкуПослеЗаписиВерсий> (необязательный)

Тип: Булево.
Если Истина - после выполнения обновления истории выполнется обработка после записи версий истории данных.
Значение по умолчанию - Истина.
Значение по умолчанию: Истина.

<АвтоУдалениеИзОбработкиПослеЗаписиВерсий> (необязательный)

Тип: Булево.
Если параметр имеет значение Истина, то после выполнения обновления истории и обработки после записи версий истории данных выполняется удаление из обработки после записи версий, дата создания которых меньше текущей на 20 дней.
По умолчанию - Истина.
Значение по умолчанию: Истина.

Описание варианта метода:

Не рекомендуется выполнять данный метод в процессе транзакции.
  • Если метод вызывается из сеанса, в котором разделитель не используется:
    • Записываются версии данных из всех областей данных.
    • Требуется право "Администрирование".
  • Если метод вызывается из сеанса, в котором разделитель используется:
    • Записываются версии данных текущей области.
    • Требуется право "Администрирование данных".

Вариант синтаксиса: Обновление истории указанных данных

Синтаксис:

ОбновитьИсторию(<Данные>, <ВыполнитьОбработкуПослеЗаписиВерсий>, <АвтоУдалениеИзОбработкиПослеЗаписиВерсий>)

Параметры:

<Данные> (необязательный)

Тип: СправочникСсылка.<Имя справочника>, ДокументСсылка.<Имя документа>, РегистрСведенийКлючЗаписи.<Имя регистра сведений>, БизнесПроцессСсылка.<Имя бизнес-процесса>, ЗадачаСсылка.<Имя задачи>, ПланВидовХарактеристикСсылка.<Имя плана видов характеристик>, ПланСчетовСсылка.<Имя плана счетов>, ПланОбменаСсылка.<Имя плана обмена>, ПланВидовРасчетаСсылка.<Имя плана видов расчета>, КонстантаКлючЗначения.<Имя константы>.
Объект или запись регистра сведений, для которого выполняется обновление истории.

<ВыполнитьОбработкуПослеЗаписиВерсий> (необязательный)

Тип: Булево.
Если установлено значение Истина, после обновления истории выполняется обработка после записи версий истории данных.
Значение по умолчанию: Истина.

<АвтоУдалениеИзОбработкиПослеЗаписиВерсий> (необязательный)

Тип: Булево.
Если параметр имеет значение Истина, то после выполнения обновления истории и обработки после записи версий истории данных выполняется удаление из обработки после записи версий, дата создания которых меньше текущей на 20 дней.
По умолчанию - Истина.
Значение по умолчанию: Истина.

Описание варианта метода:

Не рекомендуется выполнять данный метод в процессе транзакции.
Вызов метода не требует наличия у пользователя права "Администрирование".
Вызов метода может быть выполнен в сеансе, в котором используется хотя бы 1 разделитель.

Вариант синтаксиса: Обновление истории данных по данным указанных объектов метаданных

Синтаксис:

ОбновитьИсторию(<ВключитьМетаданные>, <ИсключитьМетаданные>, <ВыполнитьОбработкуПослеЗаписиВерсий>, <АвтоУдалениеИзОбработкиПослеЗаписиВерсий>)

Параметры:

<ВключитьМетаданные> (необязательный)

Тип: Неопределено, Массив.
Объекты метаданных, по данным которых выполняется обновление истории.
Если значение параметра Неопределено или пустой массив, то обновление выполняется по данным всех объектов метаданных, за исключением указанных в параметре <ИсключитьМетаданные>.
Значение по умолчанию: Неопределено.

<ИсключитьМетаданные> (необязательный)

Тип: Неопределено, Массив.
Объекты метаданных, по данным которых не выполняется обновление истории.
Значение по умолчанию: Неопределено.

<ВыполнитьОбработкуПослеЗаписиВерсий> (необязательный)

Тип: Булево.
Если установлено значение Истина, после обновления истории выполняется обработка после записи версий истории данных.
Значение по умолчанию: Истина.

<АвтоУдалениеИзОбработкиПослеЗаписиВерсий> (необязательный)

Тип: Булево.
Если параметр имеет значение Истина, то после выполнения обновления истории и обработки после записи версий истории данных выполняется удаление из обработки после записи версий, дата создания которых меньше текущей на 20 дней.
По умолчанию - Истина.
Значение по умолчанию: Истина.

Описание варианта метода:

  • Если метод вызывается из сеанса, в котором разделитель не используется:
    • Записываются версии данных из всех областей данных.
    • Требуется право "Администрирование".
  • Если метод вызывается из сеанса, в котором разделитель используется:
    • Записываются версии данных текущей области.
    • Требуется право "Администрирование данных".
При определении области данных учитываются все разделители конфигурации.

Описание:

Обновляет историю данных.

Доступность:

Сервер, толстый клиент, внешнее соединение.
Вызов метода выполняет обращение к серверу.

Пример:

ИсторияДанных.ОбновитьИсторию();

Использование в версии:

Доступен, начиная с версии 8.3.10.

МенеджерКриптографии.НачатьДобавлениеАрхивнойМеткиВремени (CryptoManager.BeginAddingArchiveTimestamp)

МенеджерКриптографии (CryptoManager)

НачатьДобавлениеАрхивнойМеткиВремени (BeginAddingArchiveTimestamp)

Доступен, начиная с версии 8.3.20.

Вариант синтаксиса: Данные подписи в возвращаемом значении

Синтаксис:

НачатьДобавлениеАрхивнойМеткиВремени(<ОписаниеОповещения>, <Подпись>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: ОписаниеОповещения.
Содержит описание процедуры, которая будет вызвана после завершения проверки подписи со следующими параметрами:
  • <ВыходныеДанные> - ДвоичныеДанные c результатом.
  • <ДополнительныеПараметры> - значение, которое было указано при создании объекта ОписаниеОповещения.

<Подпись> (обязательный)

Тип: Строка, ДвоичныеДанные, Поток, ПотокВПамяти, ФайловыйПоток.
Подпись может размещаться в файле (в этом случае указывается имя файла) или представлена как ДвоичныеДанные, Поток, ПотокВПамяти или ФайловыйПоток.

Вариант синтаксиса: Данные подписи сохраняются в файл

Синтаксис:

НачатьДобавлениеАрхивнойМеткиВремени(<ОписаниеОповещения>, <Подпись>, <ВыходныеДанные>)

Параметры:

<ОписаниеОповещения> (обязательный)

Тип: ОписаниеОповещения.
Содержит описание процедуры, которая будет вызвана после завершения проверки подписи со следующими параметрами:
  • <ВыходныеДанные> - ДвоичныеДанные с результатом.
  • <ДополнительныеПараметры> - значение, которое было указано при создании объекта ОписаниеОповещения.

<Подпись> (обязательный)

Тип: Строка, ДвоичныеДанные, Поток, ПотокВПамяти, ФайловыйПоток.
Подпись может размещаться в файле (в этом случае указывается имя файла) или представлена как ДвоичныеДанные, Поток, ПотокВПамяти или ФайловыйПоток.

<ВыходныеДанные> (обязательный)

Тип: Строка, Поток, ПотокВПамяти, ФайловыйПоток.
Имя файла или поток, в который будут выгружены данные подписи.

Описание:

Инициирует добавление в подпись архивной метки времени 3 версии.
При добавлении метки выполняется обращение к серверам меток времени, указанным в поле АдресаСерверовМетокВремени.

По завершении, будет вызвана процедура, указанная в <ОписанииОповещения>.

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.

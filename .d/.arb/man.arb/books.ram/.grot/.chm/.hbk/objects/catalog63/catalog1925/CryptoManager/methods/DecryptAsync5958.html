МенеджерКриптографии.РасшифроватьАсинх (CryptoManager.DecryptAsync)

МенеджерКриптографии (CryptoManager)

РасшифроватьАсинх (DecryptAsync)

Доступен, начиная с версии 8.3.18.

Вариант синтаксиса: Данные в двоичном виде

Синтаксис:

РасшифроватьАсинх(<ИсходныеДанные>, <Сертификат>, <ПарольДоступаКЗакрытомуКлючу>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: Строка, ДвоичныеДанные, Поток, ПотокВПамяти, ФайловыйПоток.
Зашифрованные данные.
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как ДвоичныеДанные, Поток, ПотокВПамяти или ФайловыйПоток.

<Сертификат> (необязательный)

Тип: СертификатКриптографии.
Сертификат получателя зашифрованных данных, для которого выполняется расшифровка данных.

<ПарольДоступаКЗакрытомуКлючу> (необязательный)

Тип: Строка.
Пароль для доступа к ключевому контейнеру получателя зашифрованных данных.

Описание варианта метода:

Передает расшифрованные данные в параметре типа ДвоичныеДанные.

Вариант синтаксиса: Сохранение данных в файл

Синтаксис:

РасшифроватьАсинх(<ИсходныеДанные>, <ВыходныеДанные>, <Сертификат>, <ПарольДоступаКЗакрытомуКлючу>)

Параметры:

<ИсходныеДанные> (обязательный)

Тип: Строка, ДвоичныеДанные, Поток, ПотокВПамяти, ФайловыйПоток.
Зашифрованные данные.
Данные могут размещаться в файле (в этом случае указывается имя файла) или представлены как ДвоичныеДанные, Поток, ПотокВПамяти или ФайловыйПоток.

<ВыходныеДанные> (обязательный)

Тип: Строка, Поток, ПотокВПамяти, ФайловыйПоток.
Содержит имя выходного файла или поток, в который будут сохранены расшифрованные данные.

<Сертификат> (необязательный)

Тип: СертификатКриптографии.
Сертификат получателя зашифрованных данных, для которого выполняется расшифровка данных.

<ПарольДоступаКЗакрытомуКлючу> (необязательный)

Тип: Строка.
Пароль для доступа к ключевому контейнеру получателя зашифрованных данных.

Описание варианта метода:

Помещает расшифрованные данные в файл.

Возвращаемое значение:

Тип: Обещание.
Результат выполнения обещания - расшифрованные данные типа ДвоичныеДанные или имя выходного файла с расшифрованными данными.

Описание:

Асинхронная функция. Расшифровывает зашифрованные данные.
Чтобы метод успешно расшифровал данные, необходимо чтобы:

  • В хранилище персональных сертификатов установлен сертификат, связанный с контейнером открытого ключа;
  • Сертификат указан в качестве одного из получателей зашифрованных данных (перечислен в параметре <СписокПолучателей> метода Зашифровать).

Доступность:

Тонкий клиент, веб-клиент, мобильный клиент, сервер, толстый клиент, внешнее соединение, мобильное приложение (клиент), мобильное приложение (сервер), мобильный автономный сервер.

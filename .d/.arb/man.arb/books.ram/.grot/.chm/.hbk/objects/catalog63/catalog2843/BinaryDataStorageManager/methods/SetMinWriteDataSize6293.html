МенеджерХранилищаДвоичныхДанных.УстановитьМинимальныйРазмерЗаписываемыхДанных (BinaryDataStorageManager.SetMinWriteDataSize)

МенеджерХранилищаДвоичныхДанных (BinaryDataStorageManager)

УстановитьМинимальныйРазмерЗаписываемыхДанных (SetMinWriteDataSize)

Доступен, начиная с версии 8.3.23.

Синтаксис:

УстановитьМинимальныйРазмерЗаписываемыхДанных(<МинимальныйРазмерЗаписываемыхДанных>)

Параметры:

<МинимальныйРазмерЗаписываемыхДанных> (обязательный)

Тип: Число.
Минимальный размер записываемых двоичных данных в хранилище (в байтах).

Возвращаемое значение:

Нет

Описание:

Устанавливает минимальный размер двоичных данных (в байтах), которые будут сохраняться в хранилище двоичных данных.
Все двоичные данные типа ХранилищеЗначения, кроме данных из служебных таблиц, размер которых больше или равен указанному, будут попадать в хранилище двоичных данных.
Установка минимального размера требует монопольного доступа к информационной базе (устанавливается с помощью метода УстановитьМонопольныйРежим).


Минимальное значение параметра размера двоичных данных равно 2048 байт.
При попытке установить меньшее значение, устанавливается значение 2048 байт.
Метод доступен только из сеанса, в котором не используется ни один разделитель.

Доступность:

Сервер.

Использование в версии:

Доступен, начиная с версии 8.3.23.
